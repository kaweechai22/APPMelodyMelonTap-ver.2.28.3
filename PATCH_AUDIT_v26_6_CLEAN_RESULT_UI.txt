Patch v2.26.6 - Clean Result UI

สิ่งที่ปรับ:
1. จัดหน้า Result UI ให้ตรงกับ mockup มากขึ้น
   - ปุ่มเริ่มวัดอยู่ใต้แตงโมใน Hero card
   - ปุ่มเริ่มวัดใหม่แสดงด้านบนหลังวิเคราะห์เสร็จ
   - สถานะและคุณภาพสัญญาณอยู่ใน Hero card
   - ปุ่มฟังผล / บันทึกผล / เริ่มใหม่เป็นปุ่มรองด้านล่าง
   - การ์ดผลลัพธ์เป็น Dashboard แบบกระชับ 2 คอลัมน์บนมือถือ

2. เก็บโค้ดให้สะอาดขึ้น
   - ลบฟังก์ชัน shareResult ที่ซ้ำและซ้อนอยู่ใน startSession/resetAll
   - สร้าง shareResult กลางเพียงจุดเดียว
   - เขียน startSession ใหม่ให้ flow เดิมชัดเจนและปลอดภัยขึ้น
   - เขียน resetAll ใหม่ให้ reset UI ครบและไม่กระทบระบบวิเคราะห์
   - คง ID สำคัญเดิมไว้เพื่อให้ logic เดิมทำงานได้

3. ไม่เปลี่ยนแกนวิเคราะห์เสียง
   - NUM_TAPS, trigger, feature extraction, prediction model และ AI voice summary ยังใช้ logic เดิม

4. ตรวจซ้ำและเพิ่ม sessionStarting guard เพื่อป้องกันการกดเริ่มวัดซ้ำขณะกำลังเตรียมไมโครโฟน

5. ตรวจและแก้ระบบสีตามระดับการสุก
   - ดิบ = เขียว
   - ใกล้สุก = เหลือง
   - สุกพอดี = แดง
   - สุกเกิน = เทาอ่อน
   - อาจเน่า = เทาเข้ม
   - แยก body theme ของ A/B จากเดิมที่ใช้ state-over ร่วมกัน
   - ให้รูปแตงโมเปลี่ยนทั้งสีเนื้อและสีขอบตามระดับการสุก

6. ปรับ Hero visual ตามภาพตัวอย่างระดับสุกพอดี
   - เปลี่ยนจากวงกลมสีเรียบเป็นภาพแตงโมแบบ CSS illustration
   - มีแตงโมทั้งลูก + ชิ้นแตงโมผ่าในวงแหวน
   - สีเนื้อและสีขอบเปลี่ยนตามระดับการสุกเดิม
   - ปรับ Hero layout ให้คล้ายภาพตัวอย่าง: รูปแตงโมซ้าย ผลสุก/ข้อความขวา และปุ่มหลักอยู่ใน Hero card
